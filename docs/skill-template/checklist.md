# Skill Creation Checklist

Human-readable companion to `standard.md`. Check each box before submitting a new skill.

---

## Tier Determination

- [ ] I have answered: does this skill generate code? (`Y` → L2 or L3)
- [ ] I have answered: does incorrect output cause real damage? (`Y` → L3)
- [ ] I have answered: does this skill need quality review? (`Y` → L3)
- [ ] I have answered: does this skill need >3 reference files? (`Y` → L2 or L3)
- [ ] I have picked the correct tier (if unsure, go higher)

---

## All Tiers

- [ ] Directory name is kebab-case, matches skill name
- [ ] `SKILL.md` has YAML frontmatter with `name`, `description`, `triggers`, `doNotUseWhen`
- [ ] `triggers` has at least 5 specific keywords
- [ ] `doNotUseWhen` has at least 2 concrete scenarios
- [ ] The skill's purpose is clear from the first paragraph

---

## Level 2+

- [ ] `manifest.json` exists and is valid JSON
- [ ] `manifest.json` has `triggers`, `useWhen`, `doNotUseWhen`, `allowedTools`, `riskLevel`
- [ ] `allowedTools` is not empty, does not contain `*`
- [ ] `riskLevel` is set (not just the default)
- [ ] `referenceRouting` maps at least 3 distinct user intents
- [ ] References directory has at least 1 file
- [ ] Examples directory has at least 1 bad + 1 good example

---

## Level 3

### SKILL.md
- [ ] Contains all 8 standard sections in order
- [ ] Reference routing table is present
- [ ] Dispatch table (if applicable) uses markdown table format
- [ ] Quality gates section has 🔴 Fatal, 🟡 Warning, 🔵 Suggestion subsections
- [ ] Output contract specifies exact format
- [ ] Peer skill protocol (if applicable) has Handoff, Escalation, and Review rules
- [ ] Anti-recursion guard is explicit
- [ ] Iron laws section has ≤10 items

### manifest.json
- [ ] `lifecycle` is `"candidate"` (never `"active"` for new skills)
- [ ] `tier` is `"harness-grade"`
- [ ] `requiredEvidence` lists at least 3 fields
- [ ] `peerSkills` is populated if this skill collaborates with others

### provenance.json
- [ ] `source.origin` explains where knowledge came from
- [ ] `source.references` has at least 1 URL
- [ ] `changelog` has at least the initial version entry
- [ ] `security.riskLevel` matches `manifest.json`

### checks.json
- [ ] At least 5 Fatal rules
- [ ] At least 3 Warning rules
- [ ] At least 2 Suggestion rules
- [ ] Every Fatal has a non-empty `id` and `message`
- [ ] Regex patterns are correctly escaped
- [ ] Structural checks use `checkType: "structural"` with `null` pattern

### rubric.md
- [ ] Defines 3-5 scoring dimensions, each worth 20 points
- [ ] Each dimension describes what 18-20, 14-17, 8-13, and 0-7 points looks like
- [ ] Delivery rules table is present
- [ ] Fatal>0 or Score<80 = cannot deliver

### evidence.schema.json
- [ ] `required` array has at least 3 fields
- [ ] `qualityGate` object is required with `fatal`, `warning`, `suggestion` sub-fields
- [ ] Domain-specific evidence fields are defined
- [ ] File is valid JSON Schema (can validate with a JSON Schema validator)

### examples/
- [ ] At least 1 "bad" example that triggers at least 2 Fatal checks
- [ ] At least 1 "fixed" example that triggers 0 Fatal checks
- [ ] Fixed example includes the evidence block
- [ ] Bad example has annotations explaining each violation

---

## Safety Review

- [ ] `doNotUseWhen` prevents activation in clearly wrong contexts
- [ ] `allowedTools` does not grant write/shell/network access unless the skill genuinely needs it
- [ ] If `riskLevel` is "high" or "critical", `requiresReview` is `true`
- [ ] The skill does not hardcode credentials, tokens, or API keys
- [ ] The skill does not contain executable code that modifies the system

---

## Self-Evolution Review (if generated by Recursive Evolution OS)

- [ ] `lifecycle` is `"candidate"`, not `"active"`
- [ ] `provenance.json` records the Evolution OS cycle ID
- [ ] The skill passes its own `checks.json` Fatal rules when applied to the "fixed" example
- [ ] The "bad" example triggers at least 2 Fatal checks
- [ ] The "fixed" example scores ≥ 80 against `rubric.md`
- [ ] No self-promotion to active — requires independent review
